### 1) Difference between null and undefined

null বোঝায় no value। এই মূলত বুঝায় যে কোন ভ্যারিয়েবল এর ভ্যালু নেই। অর্থাৎ আমরা হয়ত কোন ভ্যারিয়েবল ডিক্লার করেছি কিন্তু সেই ভ্যারিয়েবল এর ভ্যালু দেইনি। এইটা সাধারণত ইন্টেনশনালি ডেভেলপার রা প্রায়ই করে থাকে যখন কোন ভ্যারিয়েবল এর ভ্যালু না দেয়াই লজিক্যালি সেফ মনে হয়। null এ ভ্যালু মিসিং নয় বরং ভ্যালু না থাকাই আসলে ভ্যালু। 

অন্যদিকে undefined হচ্ছে not defined. অর্থাৎ undefined অর্থ অই ভ্যারিয়েবল এর ভ্যালু মিসিং। এইটা সাধারণত ডেভেলপাররা ইচ্ছাকৃত ভাবে করেন না। কোনো ভ্যারিয়েবল এর ভ্যালু কোন ভাবে মিসিং থাকলে এইটা দেখা যায়।



### 2) What is the use of the map() function in JavaScript? How is it different from forEach()?

JavaScript এর map হলো একটি array method যেটা একটা array এর প্রত্যেকটা উপাদানের উপর ল্যুপ চালায়, এবং একটা নতুন array রিটার্ন করে। এটা খুবই পাওয়ারফুল একটা মেথড কেননা ল্যুপ করার মাধ্যমে আমরা অনেক কোড করা থেকে বেচে যাই। 

forEach ও array এর উপর ল্যুপ চালায় তবে এটা map method থেকে আলাদা এদের রিটার্ন ভ্যালু এর দিক দিয়ে। map একটা নতুন এরে রিটার্ন করলেও forEach ডিফল্ট ভাবে কিছু রিটার্ণ করেনা। forEach এরে এর প্রতিটা উপাদানের উপর ল্যুপ চালায় আর কাঙ্ক্ষিত অপারেশন চালায়। map আমরা তখন ব্যবহার করব যখন আমাদের একটা নতুন array দরকার হবে। forEach ব্যবহার করব যখন শুধু আমাদের array এর উপর ল্যুপ চালাতে হবে, নতুন কোনো array এর প্রয়োজন পরবে না। 


### 3) What is the difference between == and ===?

== হলো loose comparison operator যেটা শুধুমাত্র ভ্যালু কম্পারিজন করে দেখে যে তারা একই কিনা কিন্ত === ভ্যালু এবং সেই ভ্যালু এর টাইপ দুইটাই চেক করে। যদি দুইটা ভ্যারিএবল এর ভ্য্যলু আর টাইপ দুইটাই ম্যাচ করে তাহলেই কেবল === true রিটার্ন করবে অন্যথায় false রিটার্ন করবে। উদাহরন দিয়ে বলা যায় যে 2 == "2" এইটা true কিন্তু  2 === "2" এইটা false


### 4) What is the significance of async/await in fetching API data?

JavaScript একটা asynchronous programming language. তার মানে হল এই যে এইটা কোনো অপারেশনে সময় লাগলে সেইখানে অপেক্ষা করে না বরং সামনের কোড কম্পাইল করা শুরু করে দেয়। কিন্তু এই asynchronous behavior মাঝে মাঝে আমাদের কোডিং কে একটু জটিল করে দেয়। আমরা সহজে ফ্লো টা ঠিক রাখতে পারি না। তাই এই asynchronous behavior কে synchronize ওয়ে তে করার জন্য async/await ব্যবহার করা হয়। ডাটা ফেচিং এ async/await ব্যবহারের কারণে আমরা ডাটা এপি আই থেকে একটা ধারাবাহিক ফ্লো তে পেতে পারি। এতে করে সেই ডাটা হ্যান্ডেলিং ও ম্যানিপুলেশন সহজ হয়; কোড অনেক ক্লিন আর রিড এবল থাকে। 


### 5) Explain the concept of Scope in JavaScript (Global, Function, Block)

JavaScript এ ভ্যারিয়েবল গুলো কিছু জায়গায় present আবার কিছু জায়গায় absent থাকে। কোন ভ্যারিয়েবল যেসব জায়গায় present থাকতে পারে সেইটা অই ভ্যারিয়েবল এর স্কোপ। এটা মূলত তিন প্রকারঃ গ্লোবাল, ফাংশন আর ব্লক

Global scope এর ক্ষেত্রে ভ্যারিয়েবল টা সম্পূর্ণ কোডেই একসেস করা যায়। টপ লেভেলে ডিক্লার করা ভ্যারিয়েবল গুলো গ্লোবাল স্কোপের হয়।
Function scope এর ক্ষেত্রে ভ্যারিয়েবল টা কোনো ফাংশনের মাঝে ডিক্লার করা হয় আর সেটার ভ্যালু শুধুমাত্র অই ফাংশনের মাঝেই খুজে পাওয়া যায়; এর বাইরে একসেস করা যায় না। 
Block scope এর ক্ষেত্রে ভ্যারিয়েবল টা কোনো ব্লকের মাঝে পাওয়া গেলেও এর বাইরে পাওয়া যায় না। ব্লক হল কার্লি ব্যাকেট এর মাঝের জায়গা। এই ভ্যারিয়েবল গুলা অই ব্লকে ব্যবহারের জন্যই ডিক্লার করা হয় আর অই ব্লক ছাড়া অন্য কোথাও পাওয়া যায় না।
